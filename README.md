# Frontend Mastery Study Plan

This repository contains a structured, topic-by-topic study plan for advancing frontend engineering skills toward senior-level proficiency.

Each topic is isolated in its own Git branch to enforce focus, prevent cognitive overload, and ensure measurable progress.

- Small code examples or implementations
- Practice exercises

---

## Workflow Rules

### 1. One Topic Per Branch
Each branch must focus only on its assigned domain. No mixing of concerns.

### 2. Completion Before Progression
Do not move to the next branch until:
- All checklist items are completed
- You can explain the topic without reference material
- You can implement core patterns from memory

### 3. Implementation-First Learning
Each topic should include:
- Minimal working examples
- Practical implementation tasks
- Real-world scenarios where applicable

### 4. Main Branch Policy
The `main` branch is used only for:
- Overview documentation (this file)
- Navigation and structure reference

No feature code should be committed to `main`.

---

## Study Progression Philosophy

This roadmap is designed around progressive complexity:

1. **Foundation Layer**
   - Responsive Design
   - CSS Architecture
   - Testing

2. **Performance + UX Layer**
   - Web Performance
   - Accessibility
   - State Management

3. **System Design Layer (Frontend)**
   - API Integration Patterns
   - Real-time Patterns
   - Monitoring & Observability

4. **Security + Reliability Layer**
   - Frontend Security
   - Advanced Security

5. **Discovery + Indexing Layer**
   - SEO for Frontend
   - Advanced SEO & Structured Data

---

## Completion Criteria (Global)

By the end of this roadmap, you should be able to:

- Build responsive, scalable frontend architectures
- Design maintainable CSS systems with predictable behavior
- Implement robust testing strategies across all layers
- Optimize performance using profiling and bundle strategies
- Secure frontend applications against common attack vectors
- Design accessible, production-ready interfaces
- Integrate APIs with modern data-fetching patterns
- Handle real-time data systems reliably
- Implement observability and monitoring in production apps
- Optimize applications for search engines and social previews

---

## Recommended Usage

- Work sequentially from `01` to `13`
- Avoid parallel branch work unless revisiting concepts
- Keep each branch self-contained and reproducible
- Use commits to reflect learning progression, not just final state

## Suggested Tooling

Not required, but recommended:
- React or equivalent UI framework
- TypeScript
- Vite or Next.js
- Testing: Vitest / Jest + Playwright
- Linting: ESLint + Prettier
- Styling: Tailwind or CSS Modules (depending on branch focus)
